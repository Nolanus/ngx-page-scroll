Namespace scrolling

This page demonstrates the interaction with running scroll animation, especially if there are multiple ones. Each scroll animation belongs to a namespace. The default namespace value is default. Whenever a new scroll animation for the same namespace is started, all running ones will be stopped. This is allows to prevent two scrolling animations acting on the same scrollView DOM element, where each interferes with the other ones. All scroll animations on the same scrollView should thus share the same namespace.

This containers scroll animation uses a custom namespace customSpace. It is therefore capable to scroll independently of the status of the default namespace animations.
